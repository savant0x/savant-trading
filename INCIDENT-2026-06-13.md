## Root Causes (Ranked by Severity)

### 1. FID-146 Verification Failure Masks All Losses
**File:** `src/execution/dex/trader.rs` (FID-146 section)
**Impact:** CRITICAL — hid every loss, let the bot trade until $0

When USDC verification fails after 3 retries, the code records "breakeven PnL" and removes
the position. The bot never learns it lost money. It continues trading with shrinking capital.

**Fix needed:** On verification failure, record the position as a LOSS (assume worst case),
halt trading, and alert the operator.

### 2. Circular Spread Filter
**File:** `src/execution/dex/trader.rs` (execute_swap, spread check)
**Impact:** HIGH — could not catch the 90% slippage on the GRT swap

The spread filter compares the effective price against `quote.price` from the 0x API.
When the 0x API returns a bad route, it also reports its own bad price as the market price.
The spread comes out as 0bps. Every bad route passes.

**Fix needed:** Compare against an independent price oracle (Kraken, CoinGecko).

### 3. No Minimum Output Validation
**File:** `src/execution/dex/trader.rs` (execute_swap)
**Impact:** HIGH — would have caught the 99.5% GRT loss before broadcast

After receiving the quote, there's no check like "I'm sending $12.57 but only getting
$0.066 worth of tokens."

**Fix needed:** Compare `buyAmount * buyTokenPrice` against `sellAmount`. Reject if
received value < 90% of sent value.

### 4. No Calldata Inspection
**File:** `src/execution/dex/zero_x.rs` (build_swap_tx)
**Impact:** MEDIUM — allowed 58.7 GRT to be sent to an unknown EOA

The bot passes through whatever calldata the 0x API returns. The API can embed
`transfer()` actions that send tokens to arbitrary addresses. The bot signs and
broadcasts without inspection.

**Fix needed:** Decode the 0x batch actions and reject if any action transfers tokens
to an address that isn't the bot's wallet or a known DEX router.

### 5. No Independent Price Oracle
**File:** Throughout the execution path
**Impact:** MEDIUM — all safety checks use 0x data exclusively

Every check (quote, spread, direction verification) uses data from the same 0x API.
If the API returns bad data, all checks pass.

**Fix needed:** Add a Kraken/CoinGecko price fetch as an independent reference.

### 6. Phantom Position Tracking
**File:** `src/execution/dex/trader.rs` (place_order)
**Impact:** LOW — position quantity doesn't match on-chain holdings

The position is recorded with the LLM's requested quantity, not the actual received amount.
This creates phantom positions that the engine tries to manage.

**Fix needed:** Record position quantity from the actual on-chain received amount.
