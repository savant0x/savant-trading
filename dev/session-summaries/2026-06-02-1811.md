## Lessons Learned

- ECHO Protocol boot sequence (Steps 1–7) is clean and deterministic when followed: read ECHO.md → config → standards → LEARNINGS → FIDs → prior session summary → create new summary.
- An "active" session summary that was never marked `completed` is a real failure mode — the lifecycle has no automatic enforcement. Recommend either a pre-commit hook that checks the most recent summary's status, or a hard rule that the previous session must be closed before a new one is opened.
- Working-tree state from a prior session can leak into the next without a `cargo check` / `cargo test` checkpoint at session boundaries. Next session should always run the full validation triad before claiming any inherited work is clean.
