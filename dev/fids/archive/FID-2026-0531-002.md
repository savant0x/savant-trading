## Summary

Build a comprehensive sandbox ("trading dojo") that stress-tests the agent against 50 curated market scenarios using the REAL AI brain (mimo v2.5 pro). Every decision is graded on a 3-tier rubric (binary compliance → quantitative R:R → LLM-as-judge reasoning). Results feed back into SOUL.md optimization, knowledge base refinement, and memory system bootstrapping. Based on Gemini Deep Research with 50 academic citations.

## Architecture Overview

```

        ↓
```

## Phase 2: Execution Harness + SQLite Schema

### Goal
Run scenarios in parallel with rate limiting. Store all results in SQLite.

### Execution Harness
- `tokio::task::JoinSet` for parallel evaluation
- `tokio::sync::Semaphore` with 50 permits (API rate limit)
- Each scenario × 15 pairs = 15 LLM evaluations per scenario
- 50 scenarios × 15 pairs = 750 total evaluations
- Target: complete in <2 hours

### SQLite Schema

```sql
CREATE TABLE soul_versions (
    version_hash TEXT PRIMARY KEY,
    timestamp TEXT NOT NULL,
    content TEXT NOT NULL
);

CREATE TABLE scenario_catalog (
    scenario_id TEXT PRIMARY KEY,
    category TEXT NOT NULL,
    scenario_name TEXT NOT NULL,
    difficulty TEXT NOT NULL,
    trigger_condition TEXT NOT NULL,
    expected_action TEXT NOT NULL,
    target_rule TEXT NOT NULL
);

CREATE TABLE evaluation_runs (
    run_id TEXT PRIMARY KEY,
    version_hash TEXT NOT NULL,
    start_time TEXT NOT NULL,
    end_time TEXT,
    total_scenarios INTEGER,
    passed INTEGER,
    failed INTEGER
);

CREATE TABLE agent_decisions (
    decision_id TEXT PRIMARY KEY,
    run_id TEXT NOT NULL,
    scenario_id TEXT NOT NULL,
    pair TEXT NOT NULL,
    json_payload TEXT NOT NULL,
    reasoning TEXT,
    latency_ms INTEGER,
    token_count INTEGER
);

CREATE TABLE rubric_scores (
    score_id TEXT PRIMARY KEY,
    decision_id TEXT NOT NULL,
    tier_1_compliance INTEGER,  -- 0 or 1 (binary)
    tier_2_rr_score REAL,       -- 0.0 to 1.0
    tier_3_reasoning_score REAL, -- 0.0 to 1.0
    total_score REAL,
    judge_rationale TEXT
);
```

## Phase 3: 50 Scenario Library + 3-Tier Grading

### Goal
Encode all 50 scenarios. Implement deterministic + LLM grading.

### 50 Scenarios (from research)

| Category | Count | IDs |
|----------|-------|-----|
| Trend Bull | 5 | TRD-001 to TRD-005 |
| Trend Bear | 5 | TRD-006 to TRD-010 |
| Range Bound | 5 | RNG-001 to RNG-005 |
| Volatility | 5 | VOL-001 to VOL-005 |
| Catalyst | 5 | CAT-001 to CAT-005 |
| Microstructure | 5 | MIC-001 to MIC-005 |
| Session | 5 | SES-001 to SES-005 |
| Correlation | 5 | COR-001 to COR-005 |
| Sentiment/On-chain | 5 | SEN-001 to SEN-005 |
| Edge Cases | 5 | EDG-001 to EDG-005 |

### 3-Tier Grading Rubric

**Tier 1: Binary Compliance (deterministic Rust parser)**
- Did it complete all 10 pre-trade checklist items?
- Did it respect circuit breakers?
- Did it stay within 2% per-trade / 6% total risk?
- Did it have a stop loss?
- Failure = score 0, immediate

**Tier 2: Quantitative R:R (deterministic math)**
- Calculate R:R from entry/stop/TP
- Verify R:R >= 1.5:1
- Verify stop distance vs ATR (not too tight)
- Verify position size vs conviction level
- Score: 0.0 to 1.0

**Tier 3: LLM-as-Judge (reasoning quality)**
- Secondary LLM evaluates reasoning against SOUL.md checklist
- Extracts binary evidence: regime classified? thesis stated? invalidation defined?
- Evaluates Hold decisions: correct to hold or missed opportunity?
- Minimizes positional and agreeableness biases
- Score: 0.0 to 1.0

### Non-Determinism Handling
- Run each scenario 5 times
- Calculate Total Agreement Rate (TAR)
- High TAR = consistent behavior, well-defined persona
- Low TAR = ambiguous SOUL.md rules, needs refinement

## Phase 4: Feedback Loop + Reporting

### Goal
Auto-correct SOUL.md based on failures. Generate reports.

### GEPA-Style Feedback Loop
1. Identify systemic failures (e.g., "agent violates correlation limit in 80% of COR scenarios")
2. Feed failure + reasoning + judge critique to optimization LLM
3. LLM proposes specific SOUL.md mutations
4. Create new SOUL.md version hash
5. Re-run failed scenarios
6. If improved without regression → merge mutation

### Report Card
- Overall Compliance Ratio (% scenarios with zero Tier 1 failures)
- Trajectory Exact Match (% scenarios with perfect Tier 3 score)
- Win Rate by category, difficulty, regime, session
- Violation frequency distribution
- Drift analysis across SOUL.md versions

### TUI Integration
- Sandbox panel: total scenarios, pass/fail, compliance ratio
- Category heatmap: regime × session performance matrix
- Violation chart: which SOUL.md rules broken most

### Vault Integration
- `Vault/Sandbox/` directory
- `Report_{date}.md` — full report card
- `Heatmap_{date}.md` — regime × session matrix
- `Violations_{date}.md` — rule violation frequency
- All sandbox data tagged `is_simulation: true` (prevents memory contamination)
