## Summary

Knowledge base has 3,069 units across 33 files with massive overlap, inconsistent condition tagging, and no clear organizational structure. The agent's condition-based selection works but the source files are chaotic — 11 `book_*.json` files organized by source (book title), 22 original transcript files organized by speaker, and zero deduplication. The agent may see duplicate knowledge from different sources or miss relevant knowledge because conditions are inconsistently tagged. This needs to be reorganized into 10 enterprise-grade files organized by TRADING FUNCTION, with proper deduplication, consistent condition tagging, and crypto-native coverage.

## Proposed Solution

### Approach

Merge all 33 files into 10 function-organized files. Deduplicate. Re-tag conditions. Add `tags` field for granular matching. Ensure crypto-native coverage in every file.

### Tags System

Every knowledge unit gets a `tags` array for precise matching beyond broad conditions:

| Tag Category | Examples | Purpose |
|---|---|---|
| **setup_type** | breakout, pullback, reversal, continuation, range-bounce, squeeze, divergence, fakeout, gap-fill | Match to current chart pattern |
| **timeframe** | scalp, intraday, swing, position, weekly | Match to intended hold duration |
| **trigger** | fomc, halving, liquidation, whale-etf, funding-spike, news-event, regulatory, hack | Match to current catalyst |
| **regime_subtype** | bull-strong, bull-weak, bear-strong, bear-weak, range-tight, range-wide, volatility-expansion, capitulation | Match to current regime |
| **asset_behavior** | large-cap, mid-cap, altcoin, defi, meme, stablecoin | Match to pair being evaluated |
| **risk_context** | drawdown, winning-streak, new-position, scaling-in, trailing-stop, revenge-risk | Match to current portfolio state |
| **indicator** | rsi-divergence, macd-cross, ema-bounce, bollinger-squeeze, volume-spike, vwap, fibonacci | Match to indicator conditions |

**Selection scoring:**
```
score = conditions_match × 3 + topic_match × 2 + tags_match × 1
```

Higher tag match = more relevant knowledge injected. Ensures agent gets "How to trade breakouts during FOMC in a bull market" instead of generic trend-following advice.

### Steps

1. **Backup** — Copy all 33 files to `knowledge/_backup/` before any changes
2. **Add `tags` field to KnowledgeUnit** — Vec<String> in `knowledge.rs`, serde `default` for backward compat
3. **Update `select()` scoring** — Score = conditions×3 + topic×2 + tags×1, sort by score descending
4. **Update KnowledgeTopic enum** — Add variants if needed for new categories (check if PriceAction, MarketRegime, CryptoNative are missing)
5. **Deduplication criteria** — When same concept appears in multiple files: (a) longest content wins, (b) if equal length, most actionable wins (has specific numbers/rules), (c) if equal actionability, most recent source wins
6. **Re-tag conditions** — Ensure every unit has proper condition tags, especially crypto-native conditions
7. **Auto-tag during merge** — Use task agents to tag each unit with setup_type, timeframe, trigger, regime_subtype, etc. during the merge process (not manual)
8. **Merge into 10 files** — Write the consolidated files with tags
9. **Delete old files** — Remove the 33 source files (backup exists in `knowledge/_backup/`)
10. **Wire Glass House knowledge projection** — After `load_knowledge_base()`, call `vault_writer.project_knowledge()` with `(id, topic, content[0..50])` tuples
11. **Verify** — Build, test, check unit count, verify condition+tag coverage, verify vault Knowledge/ populated, run sandbox
