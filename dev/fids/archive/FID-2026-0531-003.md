## Issues

### WIRE-1. Memory context not injected into AI prompt

**Files:** `src/agent/context_builder.rs`, `src/engine.rs`
**Problem:** `context.rs` defines `query_memory_context()` and `format_memory_prompt()` but neither is called. The AI prompt has no "Dynamic Memory Context" section. The agent has no knowledge of its historical performance.
**Fix:**
1. Add `memory_context: Option<String>` to `FullContext`
2. In engine.rs Phase 1, query memory context per pair
3. In `build_user_message_static()`, append memory context after on-chain analytics
4. Pass memory context through FullContext

### WIRE-2. CUSUM charts not updated on trade close

**Files:** `src/engine.rs`, `src/memory/cusum.rs`
**Problem:** `CusumChart::update(actual_rr)` is never called. Edge decay detection is inactive. The agent doesn't know when its strategies are degrading.
**Fix:**
1. Initialize `HashMap<String, CusumChart>` per pair in engine startup
2. On trade close, call `cusum_chart[pair].update(achieved_rr)`
3. If `CusumAlert::NegativeShift` returned, log warning + inject into memory context
4. Persist CUSUM state to SQLite `edge_decay_alerts` table

### WIRE-3. Brier Score not calculated

**Files:** `src/engine.rs`, `src/memory/calibration.rs`
**Problem:** `calculate_brier_score()` is never called. The agent has no confidence calibration. Progressive confidence caps are not enforced.
**Fix:**
1. On each trade close, collect (confidence, is_win) pair from episodic memory
2. Calculate Brier Score when sample_size >= 20
3. Calculate `confidence_penalty_from_brier()` and inject into memory context
4. Apply `max_conviction_for_trade_count()` as a cap on the agent's stated confidence

### WIRE-4. Experience Replay never runs

**Files:** `src/engine.rs`, `src/memory/replay.rs`
**Problem:** `query_high_conviction_losses()` and `query_missed_opportunities()` are never called. The agent never learns from its mistakes.
**Fix:**
1. On engine startup (or every 100 trades), run experience replay
2. Query HIGH conviction losses from episodic memory
3. For each loss, generate a heuristic lesson (stored in `experience_replay_lessons`)
4. Inject active lessons into memory context as "OPERATOR RULES"

### WIRE-5. Operator rules from Lessons/ vault not loaded

**Files:** `src/memory/context.rs`, `src/vault/watcher.rs`
**Problem:** `MemoryContext.operator_rules` is always empty. The Lessons/ vault directory is read by `VaultWatcher` but the content is never passed to the memory context.
**Fix:**
1. On engine startup, read Lessons/ vault files via `VaultWatcher`
2. Pass lesson content to `MemoryContext.operator_rules`
3. Inject into AI prompt as "OPERATOR RULES (override all AI reasoning)"

### WIRE-6. CUSUM state not persisted across restarts

**Files:** `src/engine.rs`, `src/memory/episodic.rs`, `src/memory/cusum.rs`
**Problem:** CUSUM charts are in-memory only. Engine restart loses all edge decay state. The `edge_decay_alerts` SQLite table exists but is never read on startup.
**Fix:**
1. On startup, query `edge_decay_alerts` for latest CUSUM state per pair
2. Rebuild `HashMap<String, CusumChart>` from persisted data
3. On CUSUM alert, persist updated state to `edge_decay_alerts` table
4. Log: "CUSUM state restored for N pairs"

### WIRE-7. TUI Memory panel not wired

**Files:** `src/tui/mod.rs`, `src/core/shared.rs`
**Problem:** TUI has no memory panel. The agent has memory context in decisions but the operator can't see it. No visibility into Brier Score, CUSUM status, progressive confidence, or replay lessons.
**Fix:**
1. Add `MemorySnapshot` to `SharedEngineData` (Brier Score, CUSUM status per pair, confidence cap, lesson count)
2. Update `MemorySnapshot` in engine loop every 10 ticks
3. Add TUI memory panel showing: Brier Score + label, CUSUM status per pair, progressive confidence cap, active replay lesson count
