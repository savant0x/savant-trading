## Summary

Current paper trader closes 100% of position at TP1 or stop loss. The design doc specifies scale-out: 50% at 1:1, 30% at 1:2, 20% at 1:3. This is a core risk management principle from Pradeep Bondi (StockBee transcript) and Fabio Valentina (Scalper transcript).

## Detailed Description

### Problem

`check_stops()` closes entire position at TP1 or stop. No partial exits. The `ScaleLevel` enum exists on `Position` but is never used.

### Expected Behavior

- At TP1 (1:1 R:R) → close 50% of position, move stop to break-even
- At TP2 (1:2 R:R) → close 30% of remaining, move stop to TP1
- At TP3 (1:3 R:R) → close remaining 20%
- Each partial exit creates its own `TradeRecord`

## Proposed Solution

### Steps

1. **Partial close logic** — In `check_stops()`, check TP2 and TP3 in addition to TP1:
   ```rust
   match pos.scale_level {
       ScaleLevel::Full => {
           if hit_tp1 { close 50%, set Scaled50, move stop to break_even }
           else if hit_stop { close 100% }
       }
       ScaleLevel::Scaled50 => {
           if hit_tp2 { close 30% of original, set Scaled80, move stop to TP1 }
           else if hit_stop { close remaining }
       }
       ScaleLevel::Scaled80 => {
           if hit_tp3 { close remaining, set Closed }
           else if hit_stop { close remaining }
       }
       ScaleLevel::Closed => {}
   }
   ```
2. **Track original quantity** — Add `original_quantity: f64` to `Position` for calculating partial sizes
3. **Partial TradeRecords** — Each partial exit creates a `TradeRecord` with the partial quantity
4. **Verify** — build + clippy + fmt
