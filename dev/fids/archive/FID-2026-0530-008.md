## Summary

No way to test strategies on historical data before going live. Every transcript emphasizes data-driven decisions. A backtester replays historical candles through the strategy + risk + execution pipeline and outputs performance metrics, allowing strategy validation without risking capital.

## Detailed Description

### Problem

The only way to test strategies is to run the live loop on Kraken data and wait. No way to replay history, no way to compare strategy variants, no way to optimize parameters.

### Expected Behavior

- Fetch N days of historical OHLC data from Kraken
- Replay candles through strategy + risk + paper execution
- Output: total trades, win rate, P&L, profit factor, max drawdown, Sharpe ratio
- CLI command: `cargo run --bin savant -- backtest --days 30`

strategy/ — all strategies implement Strategy trait

## Proposed Solution

### Steps

1. **analysis module** — Create `src/analysis/mod.rs` with `pub mod backtester;`
2. **backtester.rs** — Core function:
   ```rust
   pub async fn run_backtest(
       config: &AppConfig,
       days: u32,
   ) -> anyhow::Result<Metrics> {
       // 1. Fetch historical candles from Kraken
       // 2. Create PaperTrader with starting_balance
       // 3. Replay candles chronologically
       // 4. For each candle: calculate indicators, detect regime, evaluate strategies
       // 5. Execute signals through paper trader
       // 6. Return final metrics
   }
   ```
3. **Backtest subcommand** — In `main.rs`, parse `backtest --days N` arg, call `run_backtest()`, print metrics
4. **Fee/slippage in backtest** — Use same fee/slippage config as paper trading for realistic results
5. **Verify** — build + clippy + fmt
