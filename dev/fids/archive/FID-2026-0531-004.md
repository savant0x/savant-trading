## Items

### SPRINT-1. Multi-Timeframe Analysis

**Problem:** SOUL.md says "Primary analysis: 4H/Daily, Entry timing: 1H, Context: Weekly." Bot only fetches 5m candles. Making decisions without higher timeframe trend/structure.

**Fix:**
1. Fetch 5m + 1H + 4H candles per pair in Phase 1
2. Compute indicators on all three timeframes
3. Add `MultiTimeframeContext` struct: 5m regime, 1H trend, 4H trend, daily S/R levels
4. Inject higher timeframe data into AI user message
5. Higher timeframe takes precedence (SOUL.md rule)

**Files:** `src/engine.rs`, `src/agent/context_builder.rs`, `src/data/market_data.rs`
**Lines:** ~80
**Verification:** AI context shows "4H: Trending Up | 1H: Ranging | 5M: Pullback to support"

### SPRINT-2. WebSocket → Engine Integration

**Problem:** `websocket.rs` parses Kraken WS v2 messages but engine still REST-polls. High latency, rate limit pressure.

**Fix:**
1. Spawn WS connection as tokio task in engine startup
2. Channel `WsMessage` from WS task to engine loop
3. Engine uses WS ticker data for price updates (real-time)
4. REST fallback for candles (WS doesn't provide historical)
5. Auto-reconnection on connection drop

**Files:** `src/data/websocket.rs`, `src/engine.rs`
**Lines:** ~60
**Verification:** Log shows "Kraken WS connected" and real-time price updates

### SPRINT-3. Scan All Pairs

**Problem:** 15 hardcoded pairs. Config has `scan_all_pairs = false` flag but no implementation. Missing opportunities on pairs not in the list.

**Fix:**
1. When `scan_all_pairs = true`, fetch all USD pairs from Kraken
2. Filter by minimum volume (> $1M daily)
3. Filter by minimum market cap (top 100)
4. Rotate through filtered pairs each cycle
5. Keep 15 "core" pairs always evaluated

**Files:** `src/engine.rs`, `src/data/kraken.rs`
**Lines:** ~50
**Verification:** Log shows "Scanned 45 pairs, filtered to 28 tradeable"

### SPRINT-6. Multi-Timeframe Config Support

**Problem:** Config only has `timeframe = "5m"`. Multi-timeframe needs a config field for which timeframes to fetch.

**Fix:** Add `timeframes = ["5m", "1h", "4h"]` to `[trading]` section in `config/default.toml`. Add `timeframes: Vec<String>` to `TradingConfig` in `core/config.rs`.

**Files:** `config/default.toml`, `src/core/config.rs`
**Lines:** ~5
**Verification:** Config loads with timeframes field.

### SPRINT-7. MeanReversionStrategy evaluate_sync()

**Problem:** `evaluate_sync()` default returns `None`. Backtesting with mean reversion silently produces zero signals.

**Fix:** Add `evaluate_sync()` override to `MeanReversionStrategy` with same logic as `evaluate()` but without async.

**Files:** `src/strategy/mean_reversion.rs`
**Lines:** ~40
**Verification:** Backtest with mean reversion produces signals.

### SPRINT-8. CORS Headers for Dashboard

**Problem:** API on localhost:8080, dashboard on localhost:3000. Cross-origin requests blocked without CORS.

**Fix:** Add `CorsLayer::permissive()` to axum router. `tower-http` with `cors` feature already in deps.

**Files:** `src/api/mod.rs`
**Lines:** ~5
**Verification:** Dashboard fetches from API without CORS errors.
