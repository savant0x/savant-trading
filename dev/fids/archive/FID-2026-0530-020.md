### Item 3: WebSocket Client (`src/data/websocket.rs`)

**Problem:** Engine uses REST polling for OHLC data. REST has high latency (100s of ms), blind between polls, and triggers rate limits during high-frequency updates.

**Expected Behavior:**
- New module `src/data/websocket.rs` connects to Kraken WebSocket v2 (`wss://ws.kraken.com/v2`)
- Subscribes to: ticker, book (Level 2), trade, ohlc channels
- Heartbeat handling (ping/pong)
- Sequence number verification for order book integrity
- Auto-reconnection on connection drop
- Order book snapshot + delta updates
- Falls back to REST if WebSocket unavailable

**Affected Components:**
- New: `src/data/websocket.rs`
- Modified: `src/data/mod.rs` (add module)
- Modified: `src/engine.rs` (use WS for real-time data, REST for historical)

---

### Item 4: Backtesting Engine (`src/backtest/mod.rs`)

**Problem:** No way to validate strategies on historical data before live deployment. The knowledge base contains Walk-Forward Optimization theory but no implementation.

**Expected Behavior:**
- New module `src/backtest/` with:
  - `engine.rs` — BacktestEngine that replays historical candles through strategies
  - `metrics.rs` — Sharpe ratio, max drawdown, win rate, profit factor, expectancy
  - `walk_forward.rs` — Walk-forward optimization with rolling windows
- CLI command: `cargo run -- backtest --from 2025-01-01 --to 2025-12-31`
- Outputs: equity curve, trade log, performance metrics
- Uses existing `Strategy` trait and `IndicatorEngine`

**Affected Components:**
- New: `src/backtest/mod.rs`
- New: `src/backtest/engine.rs`
- New: `src/backtest/metrics.rs`
- New: `src/backtest/walk_forward.rs`
- Modified: `src/lib.rs` (add module)
- Modified: `src/main.rs` (add backtest CLI command)
