## Summary

Currently REST polling on 5m intervals — misses fast moves and wastes API rate limit tokens. Kraken WebSocket v2 pushes OHLCV and order book updates in real-time. Fabio Valentina's entire scalping model depends on detecting aggression in real-time order flow.

## Detailed Description

### Problem

`engine.rs` polls `get_ohlc()` every 5 minutes. Between polls, price can move significantly. No order book data. No real-time aggression detection.

### Expected Behavior

- WebSocket connection to `wss://ws.kraken.com/v2`
- Subscribe to OHLC channels for configured pairs/timeframes
- Subscribe to book channels for order flow data
- Push updates to `MarketDataStore` via channel
- Auto-reconnect on disconnect

### Evidence

```

data/ — no ws.rs module
engine.rs — REST polling only
```

### Risk Level

## Proposed Solution

### Steps

1. **ws.rs** — Create WebSocket client:
   ```rust
   pub struct WsClient {
       url: String,
       tx: broadcast::Sender<Candle>,
   }
   impl WsClient {
       pub async fn connect(&self) -> Result<(), DataError> { ... }
       pub async fn subscribe_ohlc(&self, pair: &str, interval: u32) { ... }
       pub async fn subscribe_book(&self, pair: &str, depth: u32) { ... }
   }
   ```
2. **Kraken WS v2 protocol** — Send JSON subscribe message:
   ```json
   {"method": "subscribe", "params": {"channel": "ohlc", "symbol": ["BTC/USD"], "interval": 5}}
   ```
3. **Reconnection** — On disconnect, wait 5s, reconnect, re-subscribe
4. **Engine integration** — Replace REST poll loop with WebSocket receive loop
5. **Verify** — build + clippy + fmt

### Verification

- Connect to Kraken WS, verify OHLC messages received
- Verify candles pushed to MarketDataStore
