## Summary

Current stop loss is fixed — set at entry and never moved. Fabio Valentina's model (from Scalper transcript) requires trailing stops: move to break-even after 1R profit, then trail using ATR or structure. This locks in profits on winning trades instead of giving back gains.

## Detailed Description

### Problem

`check_stops()` in `paper.rs` only checks fixed `pos.stop_loss`. No mechanism to update the stop as price moves in favor.

### Expected Behavior

- After 1R profit reached → move stop to break-even
- After 2R profit reached → move stop to 1R profit
- Trail stop at `highest_high_since_entry - atr * multiplier` for longs (opposite for shorts)

### Evidence

```
paper.rs:50-53 — stop_loss never updated after entry
risk/stop_loss.rs:31-51 — break_even_trigger() exists but never called
```

## Impact Assessment

### Affected Components

- `src/execution/paper.rs` — add trailing stop logic in `check_stops()`, track `trailing_stop` field on Position
- `src/core/types.rs` — add `trailing_stop: f64` field to `Position`
- `src/engine.rs` — pass ATR to position for trailing calculation

### Risk Level

- [x] Medium: Winning trades give back profits without this

## Proposed Solution

### Steps

1. **Position type** — Add `trailing_stop: f64` to `Position` struct. Initialize to `stop_loss`.
2. **Trailing logic in check_stops** — After checking fixed stop, also check trailing stop:
   ```rust
   // Update trailing stop
   let risk = (pos.entry_price - pos.original_stop_loss).abs();
   let profit = if matches!(pos.side, Side::Long) {
       price - pos.entry_price
   } else {
       pos.entry_price - price
   };
   if profit >= risk && pos.trailing_stop == pos.original_stop_loss {
       // Move to break-even
       pos.trailing_stop = pos.entry_price;
   }
   if profit >= risk * 2.0 {
       // Move to 1R profit
       pos.trailing_stop = ...;
   }
   ```
3. **Check trailing in stop evaluation** — `let hit_stop = price <= pos.trailing_stop` (long) or `price >= pos.trailing_stop` (short)
4. **Verify** — build + clippy + fmt

### Verification

- Simulate a trade that moves 2R in favor, verify stop moves to break-even
- Simulate a trade that reverses after 1R, verify break-even stop protects profit
