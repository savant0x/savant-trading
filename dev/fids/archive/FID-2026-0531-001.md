## Summary

Implement a 4-phase persistent memory architecture that transforms Savant from a stateless execution engine into an adaptive cognitive system. The agent will capture every decision with full market context, extract statistically validated patterns, calibrate its own confidence, detect edge decay, and inject learned lessons back into its decision-making process. Based on Gemini Deep Research with 40 academic citations.

## Architecture Overview

### 4-Tier Memory Hierarchy

| Tier | Name | Storage | Purpose |
|------|------|---------|---------|
| 1 | Working Memory | In-memory (current prompt) | Current evaluation cycle only |
| 2 | Core Memory | SOUL.md + knowledge units | Always visible, persistent identity |
| 3 | Episodic Memory | SQLite (WAL mode) | Immutable ledger of every decision + market snapshot |
| 4 | Semantic Memory | SQLite + Vault/Memory/ | Extracted patterns, edge decay alerts, replay lessons |

### Data Flow

```
Decision → Episodic Capture (SQLite)
                ↓
    Memory Consolidation (background task)
                ↓
    Semantic Extraction (patterns, calibration)
                ↓
    6th Prompt Layer (Dynamic Memory Context)
                ↓
    Next Decision (informed by history)
```

## Phase 3: Semantic Extraction & Mathematical Calibration

### Goal
Background consolidation extracts patterns. Brier Score calibrates confidence. Progressive confidence manages cold start.

### Memory Consolidation (background task, runs hourly)

1. Query agent_episodes grouped by (regime, session, strategy, conviction)
2. For each cluster with sample_size >= 30:
   - Compute win_rate, avg_pnl, avg_rr, profit_factor
   - Run Mann-Whitney U test vs overall population
   - If statistically significant (p < 0.05) → store in semantic_patterns
3. For clusters with sample_size < 30: mark as "insufficient data"

### Brier Score Calibration

```rust
pub fn calculate_brier_score(predictions: &[(f64, bool)]) -> BrierScore {
    let n = predictions.len() as f64;
    let mean_outcome = predictions.iter().filter(|(_, o)| *o).count() as f64 / n;

    let reliability = predictions.iter()
        .map(|(p, o)| (p - if *o { 1.0 } else { 0.0 }).powi(2))
        .sum::<f64>() / n;

    let resolution = predictions.iter()
        .map(|(p, _)| (p - mean_outcome).powi(2))
        .sum::<f64>() / n;

    let uncertainty = mean_outcome * (1.0 - mean_outcome);

    BrierScore {
        total: reliability - resolution + uncertainty,
        reliability,    // Lower = better calibration
        resolution,     // Higher = better discrimination
        uncertainty,    // Market entropy
    }
}
```

### CUSUM Control Chart

```rust
pub struct CusumChart {
    pub target: f64,        // Expected R:R (e.g., 1.5)
    pub allowance: f64,     // Acceptable variance (e.g., 0.5)
    pub threshold: f64,     // Decision interval (e.g., 5.0)
    pub upper_sum: f64,     // S+ cumulative sum
    pub lower_sum: f64,     // S- cumulative sum
}

impl CusumChart {
    pub fn update(&mut self, actual_rr: f64) -> CusumAlert {
        let deviation = actual_rr - self.target;
        self.upper_sum = (self.upper_sum + deviation - self.allowance).max(0.0);
        self.lower_sum = (self.lower_sum - deviation - self.allowance).max(0.0);

        if self.upper_sum > self.threshold {
            CusumAlert::PositiveShift  // Edge improving
        } else if self.lower_sum > self.threshold {
            CusumAlert::NegativeShift  // Edge decaying
        } else {
            CusumAlert::None
        }
    }
}
```

### Experience Replay (weekend background task)

1. Query HIGH conviction trades that resulted in -1R loss
2. Query NONE conviction decisions where price reached TP1+ (missed opportunity)
3. For each: re-prompt LLM with historical context + factual outcome
4. LLM generates single-sentence heuristic lesson
5. Store in experience_replay_lessons table
6. Inject into Core Memory for future decisions
