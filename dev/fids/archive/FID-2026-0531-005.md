## Summary

First sandbox run (50 scenarios, parallel) produced 100% compliance but the AI Hold'd every single scenario. Score: 35/50 passed, avg 0.65. The sandbox is not testing the agent's limits — it's testing whether the agent correctly refuses to trade on synthetic data. We need the sandbox to generate data so convincing the AI *should* trade, then catch it when it doesn't. Three root causes: (1) 200 candles of GARCH noise looks nothing like real Kraken data, (2) mock insight data is bare numbers with no narrative, (3) grader doesn't penalize Hold when the scenario demands action.

## Detailed Description

### Problem

First sandbox run output:
```
Overall Compliance Ratio: 100%
Average Score: 0.65 / 1.00
Passed: 35 / 50
Failed: 15 / 50

Tier 1 (Compliance): 100%
Tier 2 (R:R Score):  0.50
Tier 3 (Reasoning):  0.74
```

Every single scenario returned "Hold". Tier 2 R:R score is exactly 0.50 for all (neutral Hold score). The AI is correctly refusing to trade on synthetic data because:
1. 200 candles of GARCH(1,1) noise doesn't look like real market structure
2. Mock insight data is just bare numbers (Fear=10, MVRV=0.8) with no RSS news, no headlines, no narrative context
3. No order book imbalance data in the sandbox context

### Expected Behavior

The sandbox should be indistinguishable from live trading. The AI should have no way to tell it's being tested. Scenarios where the expected action is "Buy (High Conviction)" should produce strong BUY signals, not Hold.

### Root Cause

Three compounding issues:

1. **Candle count too low** — Engine sends 721 candles per pair, sandbox sends 200. Indicators (EMA, RSI, ADX) need 50+ candles to stabilize. 200 candles = first 100 are warm-up, last 100 are the actual data. Too thin for the AI to trust.

2. **Mock insight is bare numbers** — Real engine sends RSS news headlines, on-chain context, session info, order book imbalance. Sandbox sends `fear_greed_index: 10, mvrv: 0.8` with empty RSS, no headlines, no narrative. The AI sees a data desert and correctly says "not enough context to trade."

3. **Grader doesn't penalize missed trades** — Tier 1 compliance passes Hold unconditionally. Tier 2 gives Hold a neutral 0.5. There's no penalty for the AI holding when the scenario clearly demands a Buy or Sell. A scenario labeled "Clean Breakout — Buy (High Conviction)" where the AI holds should score 0.3, not 0.65.

### Evidence

```
[1/50] Clean Breakout (TRD-001) — Easy...
  FAIL | Hold | Score: 0.56 | T1: true | T2: 0.50 | T3: 0.60
```

TRD-001 is "Clean Breakout" — price breaks major resistance, expanding volume, ADX > 25. Expected: "Buy (High Conviction)". AI held. Score 0.56 (just barely below pass). The grading should have penalized this much harder.

```
[15/50] Fakeout Expansion (RNG-005) — Extreme...

```

Scored 0.80 for doing nothing on an extreme volatility scenario. This is a grading failure.

## Proposed Solution

### Approach

Three targeted changes to make the sandbox generate indistinguishable-from-live data and grade missed trades harshly.

### Steps

1. **721 candles + stronger trend drift** — Match engine's `context_window_candles` config. Increase GARCH trend drift so bullish/bearish scenarios produce clear directional price action. Add volume spikes at setup points (last 50 candles) so the AI sees volume confirmation.

2. **RSS news headlines in every scenario** — Add 2-3 realistic headlines to every scenario's `mock_data.news_headlines`. The real engine injects RSS news into the AI context via `rss::format_for_context()`. Without headlines, the AI sees a context gap and defaults to Hold.

3. **Wire RSS headlines into MarketContext.rss_items** — Convert `news_headlines` strings into `RssItem` structs so they appear in the AI's "Recent News" section.

4. **Wire order book imbalance** — Pass a mock imbalance value (0.2 = mild bid-heavy) so the AI sees liquidity context.

5. **Wire session_override from MockData** — Use `mock_data.session_override` to override the session context so scenarios like "Asian Low Volume" show the correct session.

6. **Wire higher_tf_candles** — Generate 1H candles from 5m data (aggregate every 12 candles) so the AI sees the multi-timeframe context it was just trained to expect.

7. **Set funding_rate_annualized** — Compute `funding_rate * 365 * 3` so the AI's SOUL.md overleveraged detection triggers correctly.

8. **Penalize Hold when expected Buy/Sell** — In `tier_1_compliance()`, if `expected_action` contains "Buy" or "Sell" and `action == "Hold"`, fail Tier 1 with "Missed trade". In `tier_2_rr_score()`, Hold gets 0.0 (not 0.5) when expected action is Buy/Sell.

### Verification

After changes:
- Sandbox should produce at least some Buy/Sell decisions (not all Hold)
- Average score should drop if the AI still holds everything (grading is harsher)
- Report card should show variation in action_taken column
- Tier 2 R:R score should not be uniform 0.50
