## CRITICAL — Fix First (Risk Management Broken)

### C1. `update_equity()` never called — drawdown kill switch non-functional
**File:** `src/engine.rs:325-721`
**Problem:** Main loop never calls `paper.update_prices(&prices)`. Account equity, unrealized PnL, and drawdown are always stale/zero. Circuit breaker drawdown check is meaningless.
**Fix:** After constructing `prices` HashMap, call `paper.update_prices(&prices)` before circuit breaker check. Already have `prices` at line ~627.
**Verification:** `cargo run -- --dry-run` — log shows non-zero drawdown when positions open.
**Lines changed:** ~3

### C2. Rate limiter never resets — permanent lockout
**File:** `src/api/mod.rs:302-318`
**Problem:** `REQUEST_COUNT` AtomicU64 increments forever. Once >1000, API returns 429 permanently. Also `fetch_sub` can underflow.
**Fix:** Replace with sliding window: track timestamp of window start, reset counter every second. Or use `Arc<RwLock<(u64, Instant)>>` with window reset.
**Verification:** Rapid requests to `/api/status` — counter resets, no permanent lockout.
**Lines changed:** ~20

### C3. Duplicate `parse_timeframe` — different semantics
**File:** `src/engine.rs:36-58`, `src/main.rs:162-172`
**Problem:** Two functions, same name, different return types (u64 seconds vs u32 minutes), different input formats ("1h" vs "60m").
**Fix:** Remove `main::parse_timeframe`. Add `parse_timeframe_minutes()` to engine that returns `u32` minutes. Use it in `run_backtest_cmd`.
**Verification:** `cargo run -- backtest` still works.
**Lines changed:** ~10

### C4. Entry fee not deducted — unlimited positions
**File:** `src/execution/paper.rs:300-311`
**Problem:** `place_order()` only deducts exit fee. Entry cost (`fill_price * quantity`) never reserved. Paper trader can open unlimited positions.
**Fix:** Deduct `cost + entry_fee` from balance. Check `balance >= cost + entry_fee` before opening.
**Verification:** Paper trader respects balance limits.
**Lines changed:** ~5

## HIGH — Fix Second (Dangerous Behavior)

### H5. `block_on` in TUI draw() — potential deadlock
**File:** `src/tui/mod.rs:129, 185, 247, 291`
**Problem:** `tokio::runtime::Handle::current().block_on()` inside async context. If RwLock held by engine task, deadlocks.
**Fix:** Read all shared data once per tick in the `run()` loop (before `terminal.draw()`), store in local `TuiSnapshot` struct, pass to `draw()`.
**Verification:** TUI renders without deadlock.
**Lines changed:** ~40

### H6. `block_on` in backtest inner loop — extremely slow
**File:** `src/backtest/engine.rs:136-141`
**Problem:** `rt.block_on()` called per candle inside `for` loop. Blocks thread, extremely slow.
**Fix:** Add synchronous `evaluate_sync()` method to `Strategy` trait that doesn't need async. Or pre-convert strategy to sync closure for backtest.
**Verification:** Backtest runs without blocking.
**Lines changed:** ~30

### H7. `side as u8` in logs — unreadable output
**File:** `src/engine.rs:423, 542, 617`, `src/agent/context_builder.rs:181`, `src/execution/paper.rs:333`, `src/agent/orchestrator.rs:139, 150`
**Problem:** `decision.side as u8` outputs `0` (Long) or `1` (Short) — meaningless in logs.
**Fix:** Implement `Display` for `Side` in `types.rs`: `Long => "LONG"`, `Short => "SHORT"`. Replace `side as u8` with `side` in all 7 locations.
**Verification:** Logs show "LONG"/"SHORT" instead of "0"/"1".
**Lines changed:** ~15

### H8. 5-7 AM EST session gap
**File:** `src/core/session.rs:92-103`
**Problem:** London ends 5 AM, NY starts 7 AM. Hours 5-7 classified as OffHours (0.3x). Pre-market is important for crypto.
**Fix:** Add `PreMarket` session variant for 5-7 AM EST with 0.7x multiplier.
**Verification:** `session::current_session()` returns PreMarket at 6 AM EST.
**Lines changed:** ~10

### H9. Session multiplier never used
**File:** `src/core/session.rs:55-63`
**Problem:** `Session::position_size_multiplier()` returns 0.3-1.2x but never called.
**Fix:** Call in `engine.rs` when computing position size. Multiply `risk_amount` by session multiplier.
**Verification:** Position sizes vary by session.
**Lines changed:** ~5

### H10. `daily_pnl` never reset
**File:** `src/core/types.rs:228`, `src/execution/paper.rs:91, 139`
**Problem:** `daily_pnl` accumulates from engine start. Never resets at midnight UTC.
**Fix:** Track `last_reset_date: NaiveDate` in PaperTrader. Reset `daily_pnl = 0.0` when UTC date changes.
**Verification:** `daily_pnl` resets at midnight UTC.
**Lines changed:** ~10
