## Summary

External audit agents identified critical training issues: severe short bias
(agent only shorts), 0-25% confidence bucket trading at 18% accuracy (noise),
empty vault subfolders, no formal workflow document, and protocol not synced
to v0.1.0. All must be fixed before the next training run.

## Detailed Description

### Issue 1: Confidence floor — no trade below 40% (CRITICAL)

**Evidence:** 0-25% confidence bucket: 18% accuracy, 0% avg confidence.
Agent takes trades it has zero confidence in. These should be holds.

**Fix:** In `decision_parser.rs`, after parsing, if confidence < 0.40 and
action != Hold, downgrade to Hold. One-line fix that immediately improves
edge by removing the worst trades.

### Issue 2: Short bias — agent only shorts (HIGH)

**Evidence:** Almost every trade is SHORT. Agent only goes long when funding
is deeply negative. Euphoria + funding triggers dominate.

**Root cause:** Training scenarios are dominated by bull-market data with
euphoria signals. Agent learns "always short the greed." Needs more
capitulation/bear scenarios where going long is correct.

**Fix:** Adjust random scenario generator to produce more capitulation
scenarios (MVRV < 1.0, SOPR < 1.0, Fear < 20). Currently Trend Bull,
Sentiment, Correlation get 3x weight. Need to also weight capitulation
conditions higher in `derive_expected_action()`.

### Issue 3: Vault subfolders empty (HIGH)

**Evidence:**
- `savant-vault/Insight/` — 0 files
- `savant-vault/Lessons/` — 0 files
- `savant-vault/Risk/` — 0 files
- `savant-vault/Sessions/` — 0 files
- `savant-vault/Trades/` — 0 files

VaultWriter has `project_insight()`, `project_risk_event()`,
`project_sandbox()` but they're never called from the training pipeline.

**Fix:** Wire vault calls into training batch:
- After each scenario: `project_decision()` (already exists, needs wiring)
- After each trade: `project_trade()` (already exists, needs wiring)
- After run: `project_risk_event()` with anti-pattern summary
- After run: `project_sandbox()` with report markdown

### Issue 4: Training takes too long (MEDIUM)

**Evidence:** 20 runs × 60 scenarios × ~60s avg = ~20 hours.

**Fix:** Default to 5 runs per session. `--train` runs 5 cycles.
`--train --full` runs 20 cycles. Update `run_training()` default.

### Issue 5: Workflow not formalized (MEDIUM)

**Evidence:** No document describing the closed-loop training cycle.
Each agent session reinvents the process.

**Fix:** `docs/TRAINING-WORKFLOW.md` created. Documents the complete
TRAIN → AUDIT → IDENTIFY → FIX → RETRAIN cycle.

### Issue 6: Protocol v0.1.0 not fully synced (LOW)

**Evidence:** Protocol files synced but protocol.config.yaml paths still
reference old structure.

**Fix:** Verify all paths in protocol.config.yaml match v0.1.0 template.

## Lessons Learned

1. A confidence floor is the single highest-impact one-line fix in a
   trading system. Removing bad trades improves edge more than adding
   good trades.
2. Training data bias produces agent bias. If all scenarios are bull
   euphoria, the agent learns to always short.
3. Empty vault folders mean the audit trail is broken. Wire everything.
4. Formalize workflows so agents don't reinvent processes.
