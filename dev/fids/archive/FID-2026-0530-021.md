## Items

### 1. CoinMetrics 403 — On-Chain API Fallback

**Problem:** CoinMetrics Community API returns 403 (rate-limited or geo-blocked). On-chain module returns empty data.

**Root Cause:** Free tier has aggressive rate limits. Single point of failure.

**Fix:** Add fallback chain: CoinMetrics → CoinGecko `/coins/bitcoin` (market_cap, fdv) → blockchain.info (hashrate proxy). CoinGecko's `market_cap` vs `fully_diluted_valuation` gives an MVRV-like ratio.

**Files:**
- Modified: `src/insight/onchain.rs`

**Verification:** `cargo test` — on-chain tests still pass. Manual: `--dry-run` shows MVRV data.

---

### 2. Backtest CLI Not Wired

**Problem:** README says `cargo run -- backtest` but `main.rs` has no match arm. Engine exists, no entry point.

**Root Cause:** Backtest module was built but CLI integration was deferred.

**Fix:** Add `Some("backtest")` match arm in `main.rs` that:
- Accepts `--from`, `--to`, `--strategy` args
- Loads historical candles from Kraken REST
- Runs `run_backtest()` with `BacktestConfig`
- Prints `BacktestMetrics` to stdout

**Files:**
- Modified: `src/main.rs`
- Modified: `src/backtest/engine.rs` (if any pub API adjustments needed)

**Verification:** `cargo run -- backtest` produces metrics output.

---

### 3. WebSocket Not Connected to Engine

**Problem:** `websocket.rs` parses Kraken WS v2 messages but nothing connects to `wss://ws.kraken.com`. Engine still REST-polls.

**Root Cause:** WebSocket client was built as infrastructure. Connection loop not implemented.

**Fix:**
- Add `connect()` method to `KrakenWebSocket` using `tokio-tungstenite`
- Spawn WS connection as tokio task in `engine::run()`
- Channel `WsMessage` from WS task to engine loop
- Engine uses WS data as primary, REST as fallback (if WS stale >30s)

**Files:**
- Modified: `src/data/websocket.rs` (add connect/reconnect loop)
- Modified: `src/engine.rs` (spawn WS task, consume messages)

**Verification:** `cargo run` — logs show "Kraken WS connected" and real-time price updates.

---

### 4. API + Engine Can't Coexist

**Problem:** `--api` starts the API server. Default starts the engine. They can't run together. API returns empty data without engine.

**Root Cause:** Separate code paths in `main.rs`.

**Fix:** Merge into single mode:
- Engine starts as primary tokio task
- API server starts as secondary tokio task on port 8080
- Both share `SharedEngineData`
- Remove `--api` flag (API always runs alongside engine)

**Files:**
- Modified: `src/main.rs`
- Modified: `src/engine.rs` (accept `SharedEngineData` parameter)

**Verification:** `cargo run` — engine runs AND `curl localhost:8080/api/portfolio` returns live data.

---

### 5. Knowledge Loading Path

**Problem:** Nova saw "19 of 141 units" — should be 254 from 22 files. Engine may load from CWD-relative `knowledge/` which fails if run from different directory.

**Root Cause:** `knowledge/` path is relative to CWD, not binary location.

**Fix:** Resolve knowledge path relative to binary location using `std::env::current_exe()`. Fallback to CWD. Log which path was used.

**Files:**
- Modified: `src/engine.rs` (knowledge loading logic)

**Verification:** `cargo run -- --dry-run` — logs "Loaded 254 knowledge units from 22 files".

### 6. CI Pipeline

**Problem:** No automated quality gates. Regressions can slip in.

**Fix:** GitHub Actions workflow:
```yaml

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
      - run: cargo test
      - run: cargo clippy -- -D warnings
      - run: cargo fmt --check
```

**Files:**
- New: `.github/workflows/ci.yml`

**Verification:** Push to GitHub → Actions tab shows green.

### 7. Rate Limiting on REST API

**Problem:** Axum server has no request throttling. A runaway client could hammer it.

**Fix:** Add `tower::limit::RateLimitLayer` to the router. Default: 100 requests/second per IP.

**Files:**
- Modified: `src/api/mod.rs`
- Modified: `Cargo.toml` (tower is already a transitive dep of axum)

**Verification:** Rapid requests to `/api/status` — no crashes, rate limit headers present.

### 8. TUI Dashboard (Ratatui)

**Fix:** Replace `--api` with a full Ratatui TUI:

**Layout:**
