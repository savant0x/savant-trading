## Summary

Paper trader doesn't deduct trading fees or model slippage on any fill. With $50-100 capital on Kraken (0.16% maker / 0.26% taker fees), fees significantly impact profitability. Without fee modeling, backtests and paper trading results are unrealistically optimistic.

## Environment

- **OS:** Windows 11 (win32)
- **Language/Runtime:** Rust 2021 edition
- **Commit/State:** Uncommitted — compiles clean

## Detailed Description

### Problem

`PaperTrader::place_order()` fills at exact price with zero cost. `check_stops()` exits at exact stop/TP price. No fees deducted, no slippage applied. This makes all PnL calculations wrong.

### Expected Behavior

- Every fill deducts `quantity * price * fee_rate` from balance
- Market orders apply slippage: `fill_price = price * (1 + slippage_pct)` for longs, `price * (1 - slippage_pct)` for shorts
- Config exposes `fee_rate` and `slippage_pct` under `[trading]`

### Root Cause

Fee and slippage were listed in DESIGN.md but never implemented.

### Evidence

```
paper.rs:138 — let fill_price = price.unwrap_or(0.0); // no fee/slippage
paper.rs:61-65 — exit_price = stop_loss or take_profit_1 // exact, no slippage
config.rs — no fee_rate or slippage_pct fields
default.toml — no fee_rate or slippage_pct
```

## Impact Assessment

### Affected Components

- `config/default.toml` — add `fee_rate = 0.0026` and `slippage_pct = 0.0005`
- `src/core/config.rs` — add fields to `TradingConfig` and `Default`
- `src/execution/paper.rs` — deduct fees in `place_order()`, apply slippage in `check_stops()`

### Risk Level

- [x] High: All PnL calculations are wrong without this

## Proposed Solution

### Steps

1. **Config** — Add `fee_rate: f64` (default 0.0026) and `slippage_pct: f64` (default 0.0005) to `TradingConfig`
2. **Fees in place_order** — After calculating `fill_price`, deduct fee:
   ```rust
   let fee = fill_price * quantity * self.fee_rate;
   self.account.balance -= fee;
   ```
3. **Slippage in check_stops** — Apply slippage to exit price:
   ```rust
   let slippage = exit_price * self.slippage_pct;
   let adjusted_exit = if hit_stop {
       exit_price - slippage // worse fill on stop
   } else {
       exit_price - slippage // worse fill on TP too (conservative)
   };
   ```
4. **Pass config to PaperTrader** — `PaperTrader::new(starting_balance, fee_rate, slippage_pct)`
5. **Verify** — `cargo build && cargo clippy -- -D warnings && cargo fmt --check`

### Verification

- Run paper trading, check logs show fee deductions on each fill
- Verify PnL is lower than without fees (sanity check)
