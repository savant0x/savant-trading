## Lessons Learned

- Knowledge depth matters more than breadth — extracting specific rules with numbers (1:2.5 R:R, 0.25% risk, 3 consecutive losses = stop) is more actionable than generic summaries
- Live data and knowledge are force multipliers when combined — the selector needs both conditions AND a rich knowledge pool
- Free APIs (alternative.me, CoinGecko) provide significant value with zero cost or setup
- Always verify API access before building FID — confirmed both APIs work before committing to implementation
