## Proposed Solution

### Approach

Fix all violations in a single pass organized by law number. Each fix is minimal and surgical. No new features — only compliance corrections. main.rs will be split into `src/engine.rs` (orchestrator logic) to stay under 300 lines.

### Steps

1. **Law 6 — unwrap() fixes (4 items)**
   - `indicators.rs:150` — replace `result.last().unwrap()` with `result.last()?` or guard
   - `indicators.rs:220` — replace `.unwrap()` with `.unwrap_or(std::cmp::Ordering::Equal)` or use `partial_cmp(...).unwrap_or(...)` with safe default
   - `paper.rs:221` — replace `.unwrap()` with `.unwrap_or_default()` or use `SystemTime::now().duration_since(UNIX_EPOCH).unwrap_or_default()`
   - `main.rs:28` — replace `.parse().unwrap()` with `.unwrap_or_else(|_| "savant_trading=info".into())`

2. **Law 5 — placeholder removal (1 item)**
   - `live.rs` — remove the entire `LiveTrader` struct and implementation. Remove `pub mod live;` from `execution/mod.rs`. Live trading will be re-added when actually needed.

3. **Law 14 — error handling (2 items)**
   - `events.rs:17` — log the error: `if let Err(e) = self.sender.send(event) { warn!("Event send failed: {}", e); }`
   - `paper.rs:91,185` — add guard: `if self.account.balance + pnl < 0.0 { warn!("Balance would go negative"); } self.account.balance += pnl;`

4. **Structural — dead code and imports (5 items)**

   - `types.rs:1` — change to `use chrono::Utc;` (DateTime not used directly)
   - `indicators.rs:6-8` — remove `new()` method from unit struct

5. **Structural — performance (2 items)**

7. **Structural — file length (1 item)**

## Lessons Learned

- Always follow ECHO Protocol boot sequence before building — the protocol exists to prevent exactly these issues
- Run Perfection Loop on each module before moving to the next — don't batch-compile an entire system
- Present changes before implementing (Law 2) — the user is a partner, not a rubber stamp
- Never use `unwrap()` — even in quick prototypes, because prototypes become production code
- Avoid unnecessary allocations — `clone()` on every loop iteration wastes memory and masks design issues
- Keep files under `max_file_lines` — if a file exceeds 300 lines, it needs to be split
