## Summary

The paper trading system currently runs in-memory only — all trades, positions, and account state are lost on restart. The SQLite trade journal exists in `monitor/journal.rs` with `TradeJournal::new()` and `record_trade()` already implemented but never wired into the engine. The user needs persistent tracking to evaluate strategy performance over time before committing real capital.

## Environment

- **OS:** Windows 11 (win32)
- **Language/Runtime:** Rust 2021 edition
- **Tool Versions:** cargo 1.x, rustc 1.x
- **Commit/State:** Uncommitted working tree — code compiles clean (build + clippy + fmt)

## Detailed Description

### Problem

Paper trader stores all state in memory. When the process stops, all trade history is lost. The user cannot evaluate whether the trading system is profitable over days/weeks of simulated trading.

### Expected Behavior

- All trades persist in SQLite across restarts
- Account balance restores on startup from last known state (calculated as starting_balance + sum of all closed trade PnL)
- Equity curve data recorded for charting/analysis
- Daily summary logged automatically
- Report command to view historical performance

### Root Cause

The `TradeJournal` in `monitor/journal.rs` was built (`new()`, `record_trade()`, `get_trades()` all exist) but never imported or called from `engine.rs`. The engine creates a `PaperTrader` in-memory and never connects to the database.

### Evidence

```
monitor/journal.rs — TradeJournal::new(), record_trade(), get_trades() implemented
monitor/mod.rs — pub mod journal; declared
engine.rs — no TradeJournal import, no database URL, no journal calls
config.rs — no database_url field in TradingConfig
Cargo.toml — sqlx = { version = "0.8", features = ["runtime-tokio", "sqlite"] } present
```

## Proposed Solution

### Approach

Wire existing `TradeJournal` into engine. Add equity snapshots table. Add daily summary. Add report CLI subcommand. No new dependencies — use `std::env::args` for arg parsing, existing `sqlx` for SQLite. Each step verified before moving to the next.

### Steps

1. **Config** — Add `database_url: String` to `TradingConfig` with default `"sqlite:data/savant.db"`. Add to `Default` impl.
2. **Incomplete candle fix** — Kraken's OHLC endpoint returns the current, not-yet-committed candle as the last entry. In `engine.rs`, after fetching candles, drop the last element before adding to `MarketDataStore`:
   ```rust
   let mut candles = candles_result?;
   if candles.len() > 1 {
       candles.pop(); // Remove incomplete current candle
   }
   ```
   This prevents indicator calculations on partial data.
3. **Journal wiring** — In `engine.rs`:
   - Create `TradeJournal::new(&config.trading.database_url)` on startup
   - Call `journal.record_trade(&trade)` when positions close (in the `for trade in closed` loop)
   - Calculate starting balance as `config.trading.starting_balance` (first run) or `starting_balance + sum(all_trade_pnl)` (subsequent runs)
4. **Equity snapshots** — Add `equity_snapshots` table to `TradeJournal::new()`:
   ```sql
   CREATE TABLE IF NOT EXISTS equity_snapshots (
       id INTEGER PRIMARY KEY AUTOINCREMENT,
       timestamp TEXT NOT NULL,
       balance REAL NOT NULL,
       equity REAL NOT NULL,
       drawdown_pct REAL NOT NULL,
       open_positions INTEGER NOT NULL
   )
   ```
   Add `record_snapshot()` method. Call every 10 ticks in the engine loop.
5. **Daily summary** — Add `daily_summary()` method to `TradeJournal`:
   ```sql
   SELECT date(closed_at) as day, COUNT(*) as trades,
          SUM(CASE WHEN pnl > 0 THEN 1 ELSE 0 END) as wins,
          SUM(pnl) as total_pnl
   FROM trades GROUP BY date(closed_at) ORDER BY day DESC
   ```
   Returns `Vec<DailySummary>` struct.
6. **Report command** — In `main.rs`, check `std::env::args().nth(1)`:
   - If `"report"` → call `report::print()` from `monitor/report.rs`
   - Otherwise → run normal engine loop
   - `report.rs` opens the DB, calls `get_trades()` and `daily_summary()`, prints formatted output
   - Update `monitor/mod.rs` to add `pub mod report;`
7. **Verify** — `cargo build && cargo clippy -- -D warnings && cargo fmt --check`

### Verification

- Run paper trading for a few minutes, stop, restart — verify trades persist and balance restores
- Run `cargo run --bin savant -- report` — verify output shows trade history and daily summaries
