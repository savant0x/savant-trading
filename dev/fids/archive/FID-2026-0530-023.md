## Issues

### PROD-1. Graceful shutdown — no SIGINT/SIGTERM handling
**File:** `src/engine.rs`, `src/main.rs`
**Problem:** Engine runs in infinite loop with no clean exit path. On Ctrl+C, process dies without saving state, closing positions, or cleaning up resources (DB connections, WebSocket).
**Fix:** Add `tokio::signal::ctrl_c()` handler in main.rs. On signal: set `engine_running` to false, save state to disk, log shutdown, exit cleanly. Use `tokio::select!` in engine loop to race between tick sleep and shutdown signal.
**Verification:** `cargo run` → Ctrl+C → logs "Shutting down...", state saved, clean exit.
**Lines changed:** ~25

### PROD-2. Block file safety mechanism
**File:** `src/engine.rs`, `src/risk/circuit_breaker.rs`
**Problem:** Knowledge base describes block file pattern (ai_claude_bot.json:87) but not implemented. When circuit breaker triggers, should write `savant.blocked` file that must be manually deleted to resume. Forces accountability.
**Fix:** On circuit breaker trigger: write `savant.blocked` with timestamp and reason. On engine startup: check if file exists, refuse to start until deleted. Add `--force` flag to bypass.
**Verification:** Trigger circuit breaker → file created → restart fails → delete file → restart succeeds.
**Lines changed:** ~30

### PROD-3. State persistence — positions/account lost on restart
**File:** `src/execution/paper.rs`, `src/engine.rs`
**Problem:** No mechanism to save engine state to disk. On restart, all positions, balance, daily_pnl start fresh. Paper trading positions are lost.
**Fix:** Add `save_state()` and `load_state()` to PaperTrader. Serialize to JSON at `data/paper_state.json`. Save after every trade close. Load on engine startup if file exists.
**Verification:** Run engine → open position → kill → restart → position still open.
**Lines changed:** ~40

### H5. TUI `block_on` in draw() — potential deadlock
**File:** `src/tui/mod.rs:129, 185, 247, 291`
**Problem:** `tokio::runtime::Handle::current().block_on()` inside async context. If RwLock held by engine task, deadlocks.
**Fix:** Read all shared data once per tick in `run()` loop before `terminal.draw()`. Store in local `TuiSnapshot` struct. Pass immutable ref to `draw()`.
**Verification:** TUI renders without deadlock. No `block_on` in draw path.
**Lines changed:** ~50
