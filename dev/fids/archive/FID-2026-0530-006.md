## Summary

TJR's core confluence concepts — Fair Value Gaps (FVG) and Order Blocks — are not implemented. These are key entry filters that strengthen the momentum strategy. FVGs identify price imbalances where price is likely to return. Order blocks identify institutional supply/demand zones.

## Detailed Description

### Problem

Momentum strategy only uses EMA + volume + range breakout. Missing the confluence filters that TJR teaches as essential for high-probability entries.

### Expected Behavior

- **FVG detection**: Identify gaps between candle 1 high and candle 3 low (bullish FVG) or candle 1 low and candle 3 high (bearish FVG)
- **Order block detection**: Identify the last opposing candle before an impulse move
- Strategies use these as confluence filters — signals only fire when price is at FVG or order block zone

### Evidence

```
strategy/momentum.rs — no FVG or order block logic
data/ — no pattern detection module
```

- `src/data/mod.rs` — add `pub mod patterns;`

## Proposed Solution

### Steps

1. **patterns.rs** — Create module with:
   ```rust
   pub struct FairValueGap {
       pub high: f64,
       pub low: f64,
       pub side: Side,
       pub index: usize,
   }
   pub struct OrderBlock {
       pub high: f64,
       pub low: f64,
       pub side: Side,
       pub index: usize,
   }
   pub fn detect_fvg(candles: &[Candle]) -> Vec<FairValueGap> { ... }
   pub fn detect_order_blocks(candles: &[Candle]) -> Vec<OrderBlock> { ... }
   ```
2. **FVG logic** — Bullish FVG: candle[i-2].high < candle[i].low (gap up). Bearish FVG: candle[i-2].low > candle[i].high (gap down).
3. **Order block logic** — Find last bearish candle before a bullish impulse (3+ consecutive bullish candles), or last bullish candle before bearish impulse.
4. **Confluence in momentum** — Before generating signal, check if price is near an FVG or order block zone. Add confidence boost if confluence present.
5. **Verify** — build + clippy + fmt
