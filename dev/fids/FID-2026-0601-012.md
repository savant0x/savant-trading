## Verified Issues

### Issue 1: Knowledge utility never persists — FIXED ✅

**Location:** `engine.rs` training batch

**Fix:** Replaced placeholder with actual `load_knowledge_base()` →
`update_utility_scores()` → `save_utility_scores()`. Scores update based on
tag overlap between unit tags and decision reasoning. Correct outcomes boost
(+lr), incorrect suppress (-lr*0.5). Scores persisted to
`data/knowledge_utility.json` after each batch.

**Limitation:** Tag-overlap heuristic is a proxy for relevance, not direct
causal attribution. True per-episode tracking requires `knowledge_units_used`
to be populated in episode snapshots (currently `vec![]`).

**Performance:** O(n*m) where n=2959 units, m=50 responses = ~150K string
comparisons per batch. Acceptable for training (runs once per batch).

**Verified:** Build clean, 127 tests, zero clippy.

### Issue 2: Anti-patterns always detect 0 — FIXED ✅

**Location:** `memory/anti_pattern.rs`

**Fix:** Added `detect_category_anti_patterns()` that joins
`episode_market_context.condition_tags` using `json_extract(condition_tags,
'$[0]')`. Groups by category, filters win_rate < 30%. Removed
`detect_session_anti_patterns()` (session is always "TestSession" — provides
no signal). Fixed all 3 remaining queries to use wins/losses PF ratio instead
of `COALESCE(pnl, 0)`.

**json_extract assumption verified:** `condition_tags` is always set to
`vec![sr.category.clone()]` in the training batch, so it's always a JSON
array with exactly one element. Safe.

**Verified:** Live DB shows 1 anti-pattern detected (was always 0).

### Issue 3: No dollar P&L in training output — FIXED ✅

**Location:** `engine.rs` training batch output

**Fix:** Added per-trade dollar P&L column. Calculates `risk_amount * rr` for
correct trades, `-risk_amount` for incorrect. Fixed $5 risk (10% of $50).

**Verified:** Output shows `P&L $+15.00`, `P&L $-5.00`, `P&L $+0.00`.

### Issue 4: HTTP stream errors 10-12% — FIXED ✅

**Location:** `agent/provider.rs`

**Fix:** Added 3-retry with exponential backoff (2s, 4s, 8s) to `chat_stream()`.
Retries on both stream request failures and stream parse failures. Only returns
Err after all 3 attempts exhausted. Logs warning on each retry with attempt
number and wait time.

**Verified:** Build clean, 127 tests, zero clippy.

### Issue 5: JSON parse errors — FIXED ✅

**Location:** Covered by Fix 4.

The retry logic in `chat_stream()` retries both the HTTP request and the
stream parse. If the LLM returns valid HTTP but empty/malformed JSON body,
`parse_streaming` returns `Err("Empty stream response")` when both content
and reasoning accumulators are empty. This triggers the retry. The 3-pass
decision parser handles partial/truncated responses. For truly empty responses,
the stream retry re-calls the LLM.

**Verified:** `parse_streaming` confirmed to return Err on empty response.

### Issue 6: Vault not populated from training — OPEN ❌

**Location:** `vault/writer.rs` — never called from training pipeline

`savant-vault/Trades/`, `Sessions/`, `Risk/`, `Insight/`, `Lessons/` are
all empty. Training produces no audit trail in the vault.

**Evidence:** Only `Decisions/` and `Knowledge/` have content (from live
engine). Training pipeline doesn't write to vault.

**Proposed fix:** Wire `VaultWriter::project_decision()` into training batch.
After each scenario LLM response, call `project_decision(pair, action,
confidence, reasoning)`. No `TradeRecord` needed — `project_decision()` only
requires pair, action, confidence, reasoning, all available in the training
batch. Trade journal (`project_trade`) deferred — requires exit_price/quantity
not available in training simulation.

**Existing code:** `VaultWriter::new(VaultConfig { vault_path: "savant-vault",
enabled: true })` with `project_decision()` and `project_trade()` methods.
Just needs wiring — ~20 lines.

## Lessons Learned

1. **Never defer without presenting** — ECHO Law 2. Scope reduction requires
   same approval as implementation.
2. **Verify before claiming** — Semantic PF was already fixed. Stale DB output
   made it look broken. Always check live state.
3. **Functions that exist but are never called are worse than missing functions**
   — They create the illusion of completeness.
4. **Every "batch wiring pending" comment is a bug** — It's a TODO masquerading
   as a note. Law 5: no TODOs without FID references.
5. **Tag-overlap is a proxy, not attribution** — True causal attribution
   requires tracking which knowledge units were in each episode's prompt.
