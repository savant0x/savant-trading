## Detailed Description

### Issue 1: On-chain data is dead (CRITICAL)

**Location:** `insight/onchain.rs`

CoinMetrics and CoinGecko both return 403 Forbidden. MVRV, SOPR, NUPL
— the signals that drove most training wins — are completely unavailable.

**Fix:** BGeometrics API — free, no key, `api_key=test`:
- `GET https://api.bgeometrics.com/v1/mvrv?api_key=test` → daily MVRV
- `GET https://api.bgeometrics.com/v1/sopr?api_key=test` → daily SOPR
- `GET https://api.bgeometrics.com/v1/nupl?api_key=test` → daily NUPL

**Validation:** Range-check parsed values (MVRV 0.1–10.0, SOPR 0.5–2.0,
NUPL -0.5–1.0). Reject out-of-range. If API fails, serve cached data
regardless of TTL (graceful degradation). If no cache, return None.

**Cache TTL:** 24 hours (daily data).

### Issue 2: Funding rate is wrong (CRITICAL)

**Location:** `insight/funding_rates.rs`

Kraken Futures returns `-45.797696%` per 8hr = `-50,148%` annualized.
Clearly wrong. OKX returns `0.01%` per 8hr — normal.

**Fix:** OKX as primary. Free, no key, not geo-blocked:
- `GET https://www.okx.com/api/v5/public/funding-rate?instId=BTC-USDT-SWAP`
- `GET https://www.okx.com/api/v5/public/open-interest?instType=SWAP&instId=BTC-USDT-SWAP`

**Pair mapping:** Engine uses "BTC/USD", OKX uses "BTC-USDT-SWAP". Need
mapping function: `BTC/USD` → `BTC-USDT-SWAP`, `ETH/USD` → `ETH-USDT-SWAP`.

**Kraken as fallback** with range validation (-2% to +2% per 8hr). Reject
garbage values. Cache 30min.

### Issue 3: RSS prompt bloat (HIGH)

**Location:** `insight/rss.rs`

Config says `rss_max_items = 10` but engine logs "RSS: 333 items from 15
feeds". Cap not enforced.

**Fix:** Enforce in RSS fetcher after dedup: sort by relevance_score
descending, take top `rss_max_items`. Add per-feed timeout (5s) to prevent
slow feeds blocking. CoinGecko trending already disabled (logs confirm).
Source diversity: take top 2 per source first, then sort all by relevance,
take top N.

**Cache TTL:** 15 minutes.

### Issue 4: No caching (HIGH)

**Location:** All insight modules

Every tick refetches all data. ~20 HTTP requests per tick × 12 ticks/hour =
240 requests/hour. Most data hasn't changed.

**Fix:** `data/cache.rs` with TTL-based cache per source. In-memory HashMap
with (value, timestamp) pairs. Check TTL before fetch. Graceful degradation:
if API fails, serve cached data regardless of TTL.

**TTLs:** Candles 5min, funding 30min, fear/greed 1h, on-chain 24h,
RSS 15min, liquidation 5min.

**Eviction:** Max 1000 entries. LRU eviction when full.

### Issue 5: No data summarization (HIGH)

**Location:** `insight/aggregator.rs`

Raw data dump into prompt. AI has to interpret everything.

**Fix:** `conditions_summary()` on MarketContext. Translates raw data into
actionable assessments using SOUL.md thresholds:
- Fear < 20 = "Extreme Fear — capitulation buy zone per SOUL.md §XIII"
- Fear 20-40 = "Fear — contrarian opportunity"
- Fear 60-80 = "Greed — caution, take profits"
- Fear > 80 = "Extreme Greed — euphoria, prepare to sell"
- MVRV < 1.0 = "Below realized value — rare buy signal"
- MVRV > 3.5 = "Euphoria — historically precedes major corrections"
- Funding > 0.05% = "Overleveraged longs — squeeze risk"
- Funding < -0.01% = "Overleveraged shorts — squeeze setup"

Wire in `build_context()`, append to market context section of user message.

### Issue 6: Excessive LLM retries (MEDIUM)

**Location:** `agent/provider.rs`

3 streaming retries + 1 non-streaming fallback = 4 attempts per pair.

**Fix:** 1 streaming + 1 non-streaming fallback = 2 total per pair.

### Issue 7: LLM stream failure in engine mode (HIGH)

Already fixed — streaming fallback to non-streaming + 180s timeout.

### Issue 8: Data not actionable (MEDIUM)

**Location:** `insight/aggregator.rs`

Raw RSS titles with no sentiment.

**Fix:** Keyword classification: "crash", "hack", "ban", "SEC enforcement",
"regulatory crackdown" = bearish. "surge", "adoption", "ETF approval",
"rally", "institutional" = bullish. Else neutral.

**Limitation:** "SEC approves Bitcoin ETF" contains "SEC" = false positive
for bearish. Mitigation: check for negation patterns ("approves",
"authorizes", "greenlights") before keyword match.

## Lessons Learned

1. **Data pipeline is the foundation.** The agent learned to trade on MVRV/SOPR
   signals. Now those signals are dead. Everything else is wasted without data.
2. **Cache before fetch.** New data sources must be cached from day one.
   Without cache, 240 HTTP requests/hour is unsustainable.
3. **Validate all external data.** Kraken returns -45% funding. BGeometrics
   could return garbage. Range validation prevents the agent from acting on
   bad data.
4. **Wire everything.** ECHO Law 4: zero grep results = NOT wired. The
   correlation matrix, vault writer, VirtualWallet, and conditions_summary
   all exist but are never called.
5. **Protocol alignment matters.** 16 structural differences create confusion.
   Keep the project synchronized with the canonical protocol.
