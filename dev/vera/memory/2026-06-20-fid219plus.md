## Lessons

- **Mechanical re-indent via Python brace-counter > hand-written str_replace for 90+ line blocks.** Capture inner_indent from anchor's leading whitespace, walk brace counter to find matching close, re-indent body +4 spaces. Promote to `coding-standards/rust.md`.
- **`shared.set_block(...)` MUST precede `std::fs::write("savant.blocked", ...)`.** Matches the wallet_reconciliation precedent. New `_halt` branches must follow this order.
- **Soft-skip divergent semantics are correct when upstream guard already hard-halts.** FID-154 hard-break + savant.blocked write is load-bearing. FID-155 soft-skip is defense-in-depth for unreachable path.
- **The `or_else` closure body must compile even if never invoked.** `FnOnce` requires syntactic validity. Prefer single `src.contains()` over `find().or_else(chain)` when both work.
- **Brittle-anchor regression tests should NOT depend on `chrono::Utc::now()` for file content.** Engine's `format!` runs at write-time, test runs at test-time — timestamps never match. Anchor on literals.
- **EADDRINUSE on port 3000 from stale Next.js dashboard blocks all Anvil smoke tests.** Must `kill` prior dashboard.exe before next run.

- **Used Python brace-counter for the FID-155 body re-indent.** Hand-editing 90+ lines would have had 100+ chances for a single bad indentation. The Python script kept the inner_indent symmetric and verified by running cargo check + cargo build.
